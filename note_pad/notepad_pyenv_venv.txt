# pyenv란?

'''
Python 프로젝트를 여러 개 관리하다 보면 이런 상황이 자주 생긴다.

프로젝트 A → Python 3.8 필요
프로젝트 B → Python 3.10 필요
최신 프로젝트 → Python 3.12 사용
Python 버전이 섞이기 시작하면 패키지 충돌이나 실행 오류가 발생하기 쉽다.

이럴 때 사용하는 도구가 바로 pyenv 이다.
'''

--> pyenv: 여러 Python 버전을 설치하고 자유롭게 전환할 수 있게 도와주는 Python 
		   버전 관리 도구
		   
------------------------------------------------------------------------
------------------------------------------------------------------------
# pyenv와 venv 차이


pyenv -> Python 버전 관리
venv  -> 프로젝트별 가상환경 관리

==>
pyenv -> Python 자체를 바꿈
venv  -> 같은 Python 안에서 패키지 환경만 분리

------------------------------------------------------------------------
------------------------------------------------------------------------
# Windows에서 pyenv 사용하기

1. Git 설치 확인
PowerShell에서 확인: git --version

------------------------------------------------------------------------
2. pyenv-win 설치
PowerShell에서 아래 명령 실행:
-> git clone https://github.com/pyenv-win/pyenv-win.git $HOME\.pyenv
 
설치가 완료되면 아래 경로에 pyenv가 생성된다.
-> C:\Users\사용자명\.pyenv

------------------------------------------------------------------------
3. 환경 변수(PATH) 설정
아래 두 경로를 시스템 환경 변수 PATH에 추가한다.

[ %USERPROFILE%\.pyenv\pyenv-win\bin ]
[ %USERPROFILE%\.pyenv\pyenv-win\shims ]
 
-> 설정 후 PowerShell을 재실행한다.

------------------------------------------------------------------------
4. 설치 확인
: pyenv --version
-> 정상 설치되었다면 버전 정보가 출력된다.

예시: pyenv 3.x.x

설치 확인 시 에러가 아래과 같은 에러가 발생한다면, PowerShell을 관리자 권한으로 
실행한 뒤 아래 명령 실행: Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
-> [Y] Yes  [N] No ...    Y

------------------------------------------------------------------------
5. Python 설치하기
설치 가능한 버전 확인: pyenv install --list

ex) Python 3.10 설치: pyenv install 3.10.11

------------------------------------------------------------------------
6. Python 버전 변경하기
현재 설치된 버전 확인: pyenv versions

전역(global) 버전 설정: pyenv global 3.10.11
-> 현재 PC 기본 Python 버전이 변경된다.

버전 확인: python --version
 
출력 예시: Python 3.10.11

------------------------------------------------------------------------
7. 프로젝트별 Python 버전 설정
특정 프로젝트 폴더에서만 Python 버전을 다르게 사용할 수도 있다.
-> pyenv local 3.10.11
 
그러면 해당 폴더에 .python-version 파일이 생성된다.
-> 이제 그 디렉터리 안에서는 자동으로 해당 Python 버전을 사용합니다.

------------------------------------------------------------------------
8. venv와 함께 사용하기
실무에서는 보통 pyenv + venv 조합을 사용합니다.

ex)
1) Python 버전 선택: pyenv local 3.10.11
 
2) 가상환경 생성: python -m venv venv
 
3) 가상환경 활성화: venv\Scripts\activate
